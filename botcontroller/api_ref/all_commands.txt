All API Commands — use 'describe' for core overview.

Available commands:
- describe
  Shows the core commands overview (api_ref/core_commands.txt).
  Use 'describe all' for the complete reference.
  usage: node api.js describe
  returns:
    - answer: api_description
    - commands: list

- describe all
  Shows the complete list of all available API commands (this file).
  usage: node api.js describe all
  returns:
    - answer: api_description
    - commands: list
- version
  Returns the current BotController version.
  returns:
    - answer: api_version
    - version: string
- get_status
  Returns the number of currently loaded bots in the BotController.
  returns:
    - answer: api_status
    - loaded_bots: number
    - mobility_mode: string
    - communication_mode: string
- get_status_extended
  Returns an extended BotController status including cluster bounding box.
  returns:
    - answer: api_status_extended
    - loaded_bots_total: number
    - loaded_cluster_bots: number
    - mobility_mode: string
    - communication_mode: string
    - bounding_box: object
- get_masterbot
  Returns the current Masterbot reference data used by the BotController.
  params:
    - (no parameters)
  returns:
    - answer: api_get_masterbot
    - id: string
    - name: string
    - connected: boolean
    - connection_slot: string
    - position: {x,y,z}
    - orientation: {x,y,z}
- get_scan_state
  Returns the current running state of Scan Level 1 and Scan Level 2.
  returns:
    - answer: api_get_scan_state
    - level1: object
    - level2: object
- gui_set_marker
  Draws one semi-transparent marker cube in the WebGUI.
  params:

    - y: number
    - z: number
    - size: number
    - color: red|green|blue|yellow|cyan|white
  returns:
    - answer: api_gui_set_marker
    - accepted: boolean
    - frontend_attached: boolean
- gui_clear_markers
  Clears all API marker cubes from the WebGUI.
  returns:
    - answer: api_gui_clear_markers
    - accepted: boolean
    - frontend_attached: boolean
- gui_refresh
  Requests the BotController WebGUI to reload the current cluster world from the controller.
  returns:
    - answer: api_gui_refresh
    - accepted: boolean
    - frontend_attached: boolean
- debug_move
  Enables, disables or queries the global MOVE diagnostics flag in the BotController.
  params:
    - mode: on|off|status
  returns:
    - answer: api_debug_move
    - debug_move_enabled: boolean
- safe_mode
  Enables, disables or queries the address recalibration safety level. Mode 0 disables recalibration, mode 1 recalibrates one addressed bot before bot-scoped API actions, and mode 2 globally recalibrates all bot addresses after successful structural changes. Default is 2.
  params:
    - mode: 0|1|2|on|off|status
  returns:
    - answer: api_safe_mode
    - safe_mode: number
- recalibrate_bot_address
  Recalculates the current local address of one known bot from the existing BotController world model without performing a full scan. Optional mode=minimal prefers shorter and simpler routes.
  The optional blocked parameter accepts a semicolon-separated list of
  coordinates (e.g. "4,1,1;3,1,1") that will be excluded from the route
  computation. Useful to force alternative routes around blocked positions.
  params:
    - bot_id: string
    - mode: standard|minimal (optional)
    - blocked: string (optional, format "x,y,z;x,y,z")
  returns:
    - answer: api_recalibrate_bot_address
    - old_adress: string
    - new_adress: string
    - changed: boolean
- recalibrate_bot_addresses
  Recalculates the current local addresses of all known non-master bots from the existing BotController world model. Optional mode=minimal prefers shorter and simpler routes.
  params:
    - mode: standard|minimal (optional)
  returns:
    - answer: api_recalibrate_bot_addresses
    - count: number
    - changed_count: number
- switch_bot_address
  Switches a bot's primary address to an alternative route (short, detour, first, standard).
  Useful for testing multi-route redundancy and hMB connectivity.
  params:
    - bot_id: string
    - target: short|detour|first|standard
  returns:
    - answer: api_switch_bot_address
    - old_adress: string
    - new_adress: string
    - changed: boolean
- set_bot_address
  Authoritatively sets a bot's address to an arbitrary value.
  Useful for manually assigning routes (e.g. for hMBs) or overriding
  a compromised/misrouted bot. Use recalibrate to recalculate later.
  params:
    - bot_id: string
    - adress: string (e.g. "FFRBTL")
  returns:
    - answer: api_set_bot_address
    - bot_id: string
    - adress: string
    - old_adress: string
    - changed: boolean
- diagnose_ack_route

  params:

    - vx: number|null
    - vy: number|null
    - vz: number|null
  returns:
    - answer: api_diagnose_ack_route

    - ack_retaddr: string
    - ack_target_neighbors_debug: array
    - ack_target_neighbors_live_debug: array
    - ack_stl_debug: object|null

- raw_cmd
  Sends a raw OP-Code command directly to the Masterbot or an ADC connector.
  Supports three routing modes:
    1) No connector + bot assigned to hMB → auto-routing via ADC
    2) Connector parameter set → explicit routing via that connector
    3) No connector + no assignment → legacy MasterBot (backward compatible)
  params:
    - value: string
    - connector: optional string (e.g. "C0", "C1", "C2")
  returns:
    - answer: api_raw_cmd
    - accepted: boolean
    - connector: string (if ADC was used)
    - auto_routed: boolean (if auto-routing was triggered)
    - target_bot: string (resolved bot ID)
  examples:
    - node api.js raw_cmd "FFFT#MOVE"                  → legacy (masterbot)
    - node api.js raw_cmd "FFFT#MOVE" C0               → via connector C0
    - node api.js raw_cmd "FFFT#MOVE"                  → auto-route if target bot is assigned

  examples:

- disable_mb
  Deactivates a helper MB. Primary MB (MB) cannot be disabled.
  All bots assigned to this MB are reassigned to remaining active MBs.
  params:
    - mb_id: string (e.g. "hMB1", "hMB2")
  returns:
    - answer: api_disable_mb
    - mb_id: string
    - active: boolean
    - reassigned: object
  examples:
    - node api.js disable_mb hMB2
- enable_mb
  Re-activates a helper MB. Bots in range may switch to it via proximity assignment.
  params:
    - mb_id: string (e.g. "hMB1", "hMB2")
  returns:
    - answer: api_enable_mb
    - mb_id: string
    - active: boolean
    - reassigned: object
  examples:
    - node api.js enable_mb hMB2
- check_mbs
  Pings the first configured slot of every MB/hMB via ADC to verify reachability.
  Each MB receives an INFO opcode on its connector; the response is awaited
  with a timeout (~3s). Returns online/offline and ping status per MB.
  returns:
    - answer: api_check_mbs
    - mbs: object (mb_id → { role, connector, active, online, ping_ok })
  examples:
    - node api.js check_mbs
- check_if_inactive
  Checks whether a specific coordinate hosts an inactive (OFFL) bot.
  Finds a known neighbor bot from the world model and sends a targeted
  CHECK to it. If the neighbor reports the slot as OFFL, the position
  is registered as inactive in the world model and a GUI refresh is triggered.
  params:
    - x: number
    - y: number
    - z: number
  returns:
    - answer: api_check_if_inactive
    - position: {x,y,z}
    - bot_found: boolean
    - inactive: boolean
    - neighbor_used: string (bot ID)
    - slot: string
    - status: string (OFFL/OK/EMPT/timeout)
  examples:

- diagnose_bot_address
  Walks the address path of a bot hop by hop, checks each intermediate position
  for inactive bots via CHECK#&#x3c;slot&#x3e;, registers any found as inactive in the
  world model, and triggers recalibrate_bot_addresses to reroute around them.
  Can find multiple inactive bots on the same path.
  params:
    - bot_id: string
  returns:
    - answer: api_diagnose_bot_address
    - bot_id: string
    - start_pos: {x,y,z} (position of the assigned MB/hMB)
    - hops: array of {index, slot, from, to, status}
    - inactive_found: array of {x,y,z, neighbor_used, slot}
    - recalibrate_triggered: boolean
    - message: string
  examples:
    - node api.js diagnose_bot_address SB1
- trace_move_path
  Traces the planned movement path of a bot and checks each coordinate
  for the bot's actual position. Useful after a move interruption
  (Fehlertyp 02) where ack="pending" and the bot stopped mid-move.
  Uses find_path_for_bot to get the path, then pings each coordinate
  via CHECK#slot# over a neighbor bot.
  If found: updates bot position + orientation + recalibrate.
  params:
    - bot_id: string
    - x: number (target x)
    - y: number (target y)
    - z: number (target z)
  returns:
    - answer: api_trace_move_path
    - bot_id: string
    - target: {x,y,z}
    - steps: array of {index, x, y, z, status}
    - found_at: {x,y,z} | null
    - found: boolean
    - message: string
  examples:

- verify_bot_position
  Explicitly checks if a bot is at a specific coordinate.
  Uses check_if_inactive internally. If the coordinate is occupied
  (status=OK): updates bot position + orientation (via ping) + recalibrate.
  If the coordinate is empty (status=EMPT): marks bot as inactive + recalibrate.
  No automatic side effects – always deliberate.
  params:
    - bot_id: string
    - x: number
    - y: number
    - z: number
  returns:
    - answer: api_verify_bot_position
    - bot_id: string
    - check_position: {x,y,z}
    - old_position: {x,y,z}
    - bot_found: boolean

    - message: string
  examples:
    - node api.js verify_bot_position SB1 4 1 1
- integrate_bot
  Fully integrates a partially known bot into the BotController world model.
  Useful after ping_position discovered a new bot (e.g. via add_bot_to in
  ClusterSim). Runs adc_assign_proximity + recalibrate_bot_addresses +

  After this command, the bot is indistinguishable from regular cluster bots
  (connector assigned, address calculated, standard colour set).
  params:
    - bot_id: string
  returns:
    - answer: api_integrate_bot
    - bot_id: string
    - actions: array of strings (e.g. ["adc_assigned", "address_recalibrated"])
    - message: string
  examples:
    - node api.js integrate_bot SB3
- set_active
  Sets a bot's active state in the BotController world model.
  When set to false, the bot is added to detected_inactive_bots and
  marked as inactive – it will be excluded from morph donor selection
  and its position will be treated as forbidden by the path planner.
  When set to true, the bot is removed from detected_inactive_bots.
  Automatically triggers recalibrate_bot_addresses + GUI refresh.
  params:
    - bot_id: string
    - active: true|false
  returns:
    - answer: api_set_active
    - bot_id: string
    - active: boolean
    - recalibrate_triggered: boolean
  examples:
    - node api.js set_active B49 false
    - node api.js set_active B49 true
- set_mobility

  Immobile bots are displayed in grey (aaaaaa) in the WebGUI.
  params:
